Set up cursorAssistant

Version 0.12.0 — bootstrap only here; project interview runs inside Cursor.

MCP install uses a pinned git tag (no network pipe-to-bash). Review the command in Cursor before confirming.

  1. Confirm in Cursor when prompted to add the cursorAssistant MCP server. It clones the package to ~/.local/share/cursorassistant on first run (no project interview yet).
  2. Open your project in Cursor (File → Open Folder).
  3. Reload Window (Developer → Reload Window) so plugin skills and commands load.
  4. Run setup in chat: say Set up cursorAssistant in this workspace, use /cursor-assistant:setup-workspace, or ask the Agent to call MCP tool lifecycle_configure. That runs the interview and writes .cursor/ + lockfile.

Manual bootstrap (terminal, more control)

Preferred if you want to inspect commands before running:

git clone --depth 1 --branch "v0.12.0" https://github.com/asafelobotomy/cursorassistant.git "$HOME/.local/share/cursorassistant/0.12.0"
ln -sfn "$HOME/.local/share/cursorassistant/0.12.0" "$HOME/.local/share/cursorassistant/current"

Or review then run:

curl -fsSL https://raw.githubusercontent.com/asafelobotomy/cursorassistant/v0.12.0/scripts/bootstrap-from-github.sh | bash

Then MCP only: cursorTools deeplink (requires bootstrap first)
