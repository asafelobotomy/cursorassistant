Your progress

Steps

  1. Confirm in Cursor when prompted to add the cursorAssistant MCP server. Package installs to ~/.local/share/cursorassistant/0.13.1 (symlink current).
  2. Open your project (File → Open Folder) — not the cursorAssistant repo unless you are developing the package.
  3. Reload Window (Command Palette → “Developer: Reload Window”) so plugin commands load.
  4. Run setup in chat — paste the phrase you copied, or use /cursor-assistant:setup-workspace, or MCP tool lifecycle_configure.

Troubleshooting
  • Cursor did not open — install Cursor, then click Install again. Or use manual bootstrap below.
  • MCP dialog shows unexpected commands — compare with deeplinks.json on GitHub for tag v0.13.1.
  • Setup command not found — Reload Window; ensure plugin symlink exists under ~/.cursor/plugins/local/cursor-assistant.
  • Already installed globally — skip the button; open your project and run the setup phrase only.
Manual bootstrap (terminal)

Inspect commands before running. Same pin as the MCP button:

git clone --depth 1 --branch "v0.13.1" https://github.com/asafelobotomy/cursorassistant.git "$HOME/.local/share/cursorassistant/0.13.1"
ln -sfn "$HOME/.local/share/cursorassistant/0.13.1" "$HOME/.local/share/cursorassistant/current"
